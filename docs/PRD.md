---

## 产品现状 (v0.1)

**已实现：**
- CCR Agent 每日 12:00 CST 自动运行
- AI 分析多源信号，生成结构化 JSON
- Next.js 静态站点，Vercel 自动部署
- 按日期归档浏览
- 类别筛选（多分类）+ 关键词搜索
- 每个机会：标题、摘要、描述、痛点、执行路径、启动成本、竞争度、收益模式、信源

**已知不足：**
- 无 SEO 优化（无 OG 图、无 sitemap）
- 无分发渠道（无邮件、无微信、无 RSS）
- 无信任积累（用户无法反馈有效性）
- 无增长机制（无分享、无推荐、无落地页）
- AI 质量依赖 prompt，尚未精调

## v1.0 目标：可公开传播的产品

### F1. SEO & 分享优化（优先级：P0）

**需求背景：** 当前分享出去是裸 URL，无预览图。这是传播的最大障碍。

**功能点：**
- [ ] 每日 OG Image 动态生成（`/api/og?date=2026-05-25`，Vercel Edge + `@vercel/og`）
  - 内容：日期 + 今日机会数量 + 精选2个标题
- [ ] `<head>` meta tags：`og:title`, `og:description`, `og:image`, `twitter:card`
- [ ] `/sitemap.xml` 自动生成（所有历史日期）
- [ ] `/robots.txt`
- [ ] 每个机会卡片有锚点链接（`#opp-{id}`），可精确分享某一条

**验收标准：** 分享到微信/微博时显示预览卡片，标题 + 图片 + 摘要完整。

---

### F2. 订阅系统（优先级：P0）

**需求背景：** 用户每天主动访问是奢望。必须推送到用户而不是等用户来。

**方案选择：** 邮件订阅（Resend + React Email）；后期扩展微信服务号

**功能点：**
- [ ] 落地页订阅入口（Header 区域 + 首页 CTA）
- [ ] 邮件模板：每日摘要（当日3–5个机会标题 + 链接）
- [ ] 触发时机：CCR Agent 写入 JSON 后，同步触发邮件发送
- [ ] 退订机制（法规合规）
- [ ] 订阅数展示（社交证明）

**依赖：** Resend 免费计划（3000封/月），存储订阅用 Vercel KV 或简单的 Notion Database。

---

### F3. 机会质量反馈（优先级：P1）

**需求背景：** AI 生成内容质量未知，需要用户投票来校准，也增加互动。

**功能点：**
- [ ] 每个机会卡片底部：👍 有价值 / 👎 不靠谱（无需登录，IP限制）
- [ ] 投票数据写入 Vercel KV
- [ ] 投票结果展示（如 "82% 认为有价值"）
- [ ] 后台：看哪类机会评分高，用于调整 CCR prompt

**验收标准：** 不登录可以投票，刷新后投票记录保留。

---

### F4. 分享单条机会（优先级：P1）

**需求背景：** 单条好机会的社交传播价值远大于整页分享。

**功能点：**
- [ ] 每个机会卡片有「分享」按钮
- [ ] 点击生成含机会信息的海报图（Canvas 或 `@vercel/og`）
  - 包含：标题、分类色条、一句话摘要、二维码/URL
- [ ] 可直接下载 PNG 或复制链接

---

### F5. CCR Prompt 精调（优先级：P0）

**需求背景：** 内容质量是产品核心竞争力。当前 prompt 是 v1 草稿，需要优化。

**优化方向：**
- [ ] 增加信号源：添加 Hacker News "Show HN"、IndieHackers、阮一峰周刊、Twitter/X 副业相关话题
- [ ] 机会筛选标准明确化：个人可执行（非融资项目）、3个月内可见收益、启动成本可控
- [ ] 质量控制：要求 AI 对每个机会评估「真实性证据」（引用具体数据或案例）
- [ ] 去重机制：检查最近 14 天历史，避免同类机会重复

## 非目标（明确不做）

- 不做用户 UGC 投稿（内容质量难以控制）
- 不做 App（Web 优先，PWA 可后期考虑）
- 不做企业版（聚焦个人执行者）
- v1.0 不做评论系统（开发成本高，优先级低）
